# plugin/semantics — 中文运维语义层

远程插件的**增值内容资产**——谷歌翻译做不到的部分。翻译只把"Host connection and power state"
译成"主机连接和电源状态";这里进一步告诉运维**症状是什么、可能为什么、该怎么处置**。

## 文件

| 文件 | 内容 |
|------|------|
| `alarm-explanations.zh-CN.json` | 告警中文解释库:告警英文名 → 中文名 / 症状 / 可能原因 / 建议处置。当前 18 条种子,随真机告警样本扩充 |

## 数据结构

```json
"<vCenter 告警英文名>": {
  "name": "中文名",
  "severity": "info | warning | critical",
  "symptom": "症状一句话",
  "causes": ["可能原因 1", "可能原因 2"],
  "actions": ["建议处置步骤 1", "步骤 2"]
}
```

## 怎么用(接入远程插件)

- 插件「告警」视图取到当前 vCenter 告警的英文名 → 在本库精确匹配 → 渲染中文名 + 症状 + 原因 + 建议
- 命中不到的告警:回退显示术语词表的中文名(`../i18n/glossary.zh-CN.json`),并记录待补
- **只读优先**:建议处置只给操作指引,不自动执行(写操作属第三阶段)

## 扩充来源

1. 真机告警样本(最准:实际触发了哪些告警)
2. vSphere/VCF 官方告警定义清单
3. SME 复核 causes/actions 后定稿
