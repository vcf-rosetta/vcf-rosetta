# vcf-rosetta R1 验证成果总结(一页纸)

> 受众:同事 / 决策者 · 日期:2026-06-22 · 环境:VCF 9.1(vc.knight.com)

## 一句话

**验证了一个商业前提:VCF 9.1 官方已删除中文,但我们可以用第三方插件把中文加回去,并且 vCenter 会正常渲染——技术上完全可行。**

## 背景(为什么做这件事)

- VCF / vCenter **9.0 起官方移除了简体中文、繁体中文、韩、德、意、巴葡**,只保留 EN + 日/西/法 4 种。
- 9.0 之前用户能用 `?locale=zh_CN` 切中文,**9.0 之后这条路也断了**——中文用户在 VCF 9 上没有任何中文界面。
- 机会:做一个 vSphere Client 插件,为中文(及后续其他语言)用户补回本地化界面 + 中文运维语义层。

## R1 要回答的问题

> 官方都删了中文,我们插件**自带**的中文,vCenter 到底认不认、显不显示?
> 这是整个项目的地基——不成立,后面都白做。

## 验证结果:✅ 全部通过

| 验证点 | 结果 |
|--------|------|
| 插件能否部署进 VCF 9.1 | ✅ 成功(状态 DEPLOYED) |
| vCenter 是否渲染我们自带的中文菜单标签 | ✅ 是("语言路由验证"正常显示) |
| 插件界面内中文是否正常 | ✅ 正常 |
| 能否自动跟随中文环境 | ✅ 能(`navigator.language = zh-CN`) |

**结论:中文可以回到 VCF 9 的界面。项目前提成立,风险 R1 关闭。**

## 这意味着什么(商业角度)

- **技术可行性已证明**:不再是"能不能做",而是"做不做、怎么卖"。
- **路径清晰**:第三方插件是 9.x 官方支持的扩展方式,我们走的是正路,不是 hack。
- **可扩展到多语言**:同一套机制,加一个语言 = 加一份资源,德语(DACH 市场)是下一个候选。

## 还需注意(诚实风险)

- 付费意愿仍需单独验证:中国是最大中文市场,但也是逃离 VMware 最快的市场;真正肯付费的可能是**被锁死的大型企业 + DACH 德语市场**。
- 本次在 **9.1** 验证,**9.0 待复测**(同流程,预期一致)。
- 当前是最小验证插件;生产插件(实际运维功能)是下一阶段工作。

## 投入产出

- 投入:几天时间,纯软件,零额外采购。
- 产出:一套**已在真机跑通**的插件工具链(打包 / 注册 / 证书 / 部署)+ 完整踩坑文档,生产阶段可直接复用。
